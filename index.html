Aula 1: História da Web

(Voltar ao início)

A história da internet é mais antiga e complexa do que muitos imaginam. Para chegar na Web que temos hoje, temos que passar pela criação dos primeiros sites e dos primeiros computadores. Alguns nomes importantes como Joseph Licklider e Grace Hopper merecem ser mencionados, por seu significado na história da tecnologia e internet como um todo.

Para explicarmos a criação da internet, temos que entender quando e porque os computadores foram criados. O primeiro computador da história foi o Harvard Mark I. Criado em 1944, o Mark I pesava mais de 4000kg e tinha centenas de quilômetros de fio.

"Mas para que ele servia??"
O Mark I servia para fazer calculos! Sim, o primeiro computador da história era uma gigantesca e cara... Calculadora!! A calculadora eletro-mecânica Mark I foi usada para fazer calculos mais complexos durante a segunda guerra mundial. Entre os criadores, estão o estudante que projetou a máquina, Howard Aiken, o designer da máquina, Norman Bel Geddes e por último mas não menos importante, uma das programadoras da máquina, que popularizou o termo "bug" na computação, Grace Hopper.

A criação da internet foi feita à partir de ideias militares, pois, no início, a internet era um meio de comunicação militar, não disponível ao público. Mais tarde, começaram testes de comunicação em pacotes enviados um a um. A primeira pessoa a pensar no conceito de uma rede de comunicações global foi Joseph Licklider. Após a guerra fria, a tecnologia foi avançado muito rapidamente, e as empresas começaram a criar seus próprios meios de comunicação, até que Tim Berners-Lee decidiu juntar hipertexto com TCP/IP, criando assim, o primeiro navegador, chamado World Wide Web (WWW). Desde então, começou uma corrida de navegadores, passando pelo HTTP e HTML, que foram popularizados por Tim Berners-Lee, o Netscape de Marc Andreessen, o Internet Explorer da Microsoft e Bill Gates, entre vários outros. E assim, foram criados os navegadores e internet que conhecemos hoje, as primeiras redes intercontinentais, entre América do Norte e Europa, foram estabelecidas por meio de cabos gigantes submersos no oceano, e acredite ou não, eles existem até hoje e ainda funcionam.

Aula 2: Relação Client x Server

(Voltar ao início)

A relação entre cliente e server é a relação entre o cliente que utiliza o site e o programador, ou owner do site, que hospeda ele em alguma máquina (servidor), trocando informações e arquivos através da internet Web, dentro do site.

O cliente pode estar em vários dispositivos diferentes, como em um computador, notebook, tablet, celular... Por este motivo, é muito importante o site ser compatível com vários desses dispositivos. Todo site é acessado por meio da internet, que faz a conexão entre a máquina do cliente e o servidor do site, para que o servidor envie as informações e arquivos de volta para o cliente.

O servidor pode armazenar conteúdos como textos, imagens, arquivos, áudios, entre outros. O servidor é onde o site armazena essas informações e arquivos, esperando algum cliente acessar o site para enviá-las para o cliente.

Os navegadores são programas criados por empresas feitos para executar arquivos de sites. Entre eles, os mais famosos com certeza são o Google Chrome, Internet Explorer, Ópera GX, entre outros.. Esses navegadores possuem funcionalidades e interpretações diferentes, mas todos seguem o básico do padrão W3C. Os primeiros navegadores foram o MOSAIC e o Netscape, a partir desses, outros foram criados e melhoraram o jeito das pessoas de navegar na internet. Completamente gratuitos, servem como um meio para a população (clientes) acesar os sites (servidores).

As aplicações WEB são sites feitos para que os clientes não precisem baixar um aplicativo para usar aquele produto. Por exemplo, o YouTube tem seu aplicativo, que é usado dispositivos móveis, em sua maioria. Porém, o YouTube também tem seu site, que pode ser usado sem baixar. Neste exemplo, o aplicativo do YouTube não é uma aplicação Web. Já o site do YouTube, esse sim é uma aplciação Web. Muitos sites tem aplicativos ou vice-versa, como o Twitter, Facebook, LinkedIn, etc..

A relação entre clientes e servidores pode ser intermediada entre várias outras funcionalidades e máquinas, como o Proxy e o Firewall.
O Proxy é um intermediário entre o cliente e o servidor dentro de uma redes de computadores (como em uma empresa), onde o cliente solicita um arquivo ou informação e o Proxy busca e se encontrar, fornece essa informação de volta para o cliente.
O Firewall é um bloqueador que está entre a máquina do cliente e o servidor, bloqueando sites e arquivos que o cliente solicita, pode ser automático como um Anti-vírus, ou mais específico, como um Firewall empresarial, que bloqueia sites improdutivos no ambiente de trabalho.


Existem alguns tipos de servidores diferentes, que servem para coisas diferentes. Entre eles estão:
  • Database server
  • DNS (Domain Name Server)
  • E-mail server
  • Web server

Aula 3: Linguagens de programação

(Voltar ao início)

Linguagem de programação é uma linguagem escrita que junta informações e cria instruções para gerar programas (softwares)

    Principais linguagens de programação atualmente

  1. Javascript
  2. Python
  3. Java
  4. PHP
  5. C#

Outro tipo de linguagem é a linguagem Server-Side, que é executada dentro do servidor. Por exemplo, as linguagens php, javascript, C#...

Ao contrário das linguagens Server-Side, temos a linguagem Client-Side, que é executado na máquina do cliente. A linguagem Javascript também roda do lado do cliente, exibindo informações e arquivos para o cliente.

Para não restar dúvidas, HTML não é uma linguagem de programação. É um arquivo de marcação, textos que o navegador interpreta e envia para o cliente.

Aula 4: Introdução à HTML

(Voltar ao início)

HTML é um arquivo de marcação, interpretado pelo navegador para entregar informações e arquivos para o cliente através do site. Pode ser escrito em qualquer programa de texto, desde um bloco de notas, Notepad++ e também o Visual Studio Code.

O Inspetor de Elementos é uma ferramente que tem em seu navegador e serve para analisar os componentes e a estrutura do site. Essa ferramenta pode ser acessada pelo atalho cntrl+shift+i.

A Estrutura Básica de HTML é "head", que é o cabeçalho da página, e é a primeira coisa que aparece para o cliente e a primeira coisa a ser carregada. Dentro do cabeçalho, temos o "title", o nome que aparece na aba do site.

Dentro do HTML, temos as tags:

  • h1, h2, h3...

    • As tags h1, h2... etc, são tags de título, h1 coloca um título principal, h2 um subtítulo (um pouco menor) e por daí em diante.

  • Mark

    • A tag Mark grifa uma palavra ou frase

  • Small

    • A tag small cria um texto menor que o texto principal.

  • i

    • A tag i deixa o texto em itálico

  • strong

    • A tag strong é usada para deixar o texto em negrito

  • ol

    • A tag ol cria uma Lista Ordenada

  • ul

    • A tag ul cria uma Lista Não Ordenada

  • li

    • A tag li cria um item na lista

  • a

    • A tag cria uma ancoragem de página

  • hr

    • A tag hr cria um divisor de linha na página

  • blockquote

    • A tag blockquote cria um texto de citação
